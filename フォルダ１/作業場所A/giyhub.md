
# GitHubの使い方

## 基礎的な定義
GitHub（ギットハブ）は、ファイルやプログラムの「保存場所」を提供してくれるウェブサービスです。複数人でファイルを安全に共有したり、作ったものの変化（履歴）を自動で記録できます。

## なぜ必要か（背景）
昔は、ファイルのバージョン管理や協力作業を手動でしていたので、間違えて新しいものを上書きしたり、誰がどこを変えたか分からなくなったりしていました。  
GitHubを使えば、誰がいつどこを直したのか追跡でき、過去の状態にも簡単に戻せます。  
世界中の人が安心して協力できるための仕組みです。

## 重要用語の整理
- **リポジトリ(Repository)：** フォルダのようなもの。中にファイルや履歴が入っている。
- **コミット(Commit)：** ファイルを「これで保存！」と記録すること。
- **プッシュ(Push)：** 自分のPCの変更をGitHubにアップロードすること。
- **プル(Pull)：** GitHubから最新のファイルをダウンロードすること。
- **ブランチ(Branch)：** 元のファイルを残しつつ、別の作業をするためのコピー。
- **プルリクエスト(Pull Request)：** 変更したところを他の人に「使っていい？」と提案すること。

## 図解イメージ
- リポジトリ  
  ├─ ファイルA  
  ├─ ファイルB  
  └─ コミット（保存履歴）
- 作業の基本の流れ  
  1. 自分のPCで編集  
  2. コミット（保存）  
  3. プッシュ（GitHubにアップ）  
  4. 他の人がプル（GitHubから取得）

## 例え話
GitHubは「みんなで編集できるノート」。それぞれがページを書き足せて、いつ誰がどこを書き換えたか分かるし、消しゴムで戻すことも簡単です。ページを丸ごとコピーして別の話を書き加え（ブランチ）、良ければ合体（マージ）させたりもできます。

## 仕組み・メカニズム
1. 手元でファイルを編集
2. 変更を「コミット」→編集の履歴を増やす
3. 「プッシュ」でGitHubにアップ
4. 他の人は「プル」でこれを使える
5. チーム作業は「ブランチ」や「プルリクエスト」を活用

## 書籍の説明をわかりやすく言い換え
「GitHubはプログラムや文書の“履歴付きの貯蔵庫”です」  
その中で、誰がいつどこをどう直したか自動で記録。複数人が同時に編集しても、それぞれの修正をちゃんと整理・統合できるので、安心して共同作業ができます。

## 初心者がつまずく点
- 「コミット」「プッシュ」「プル」の違いが混乱しやすい  
- ローカル（手元のPC）とリモート（GitHub上）の違い
- ブランチやプルリクエストの使いどころ  
- 最初は画面の専門用語や操作が難しそうに見える

2. あなたがファイルを編集してGitHubに反映させるには、どんな操作をしますか？

### 💡 pull（プル）と pull request（プルリクエスト）の違い

**pull（プル）**
- これは「GitHubから自分のパソコンに最新の状態をダウンロードする」操作です。
- チームの他の人が作業した内容を、自分も使えるように取り込むときに使います。
- イメージ：みんなで書いているノートの最新ページを、自分の手元のノートに「写す」こと。

**pull request（プルリクエスト）**
- これは「自分が直したところ・作った変更を、みんなのノートに加えていいか提案する」ものです。
- 「自分がこんな風に直しました。みんな入れて良いか確認して！」という申請。
- 他の人が内容をチェックして、OKならみんなのノート（リポジトリ）に取り込まれます。
- イメージ：自分専用で書いた新しいページを、「みんなのノートに追加して良い？」と先生や友達に見せて了承をもらう感じ。

#### まとめ（表）

| 　　　　　　　| pull（プル）                                          | pull request（プルリクエスト）               |
|-----------------|:--------------------------------------------------:|:--------------------------------------------:|
| 誰が使う？        | 主に新しい変更をもらう人                             | 主に新しい提案・修正を送る人                  |
| 何をする？        | 最新の状態を自分に取り込む操作                       | 自分の作業をみんなに提案して見てもらう操作    |
| タイミング       | 他の人が変更した後、自分もそれを取り込みたいとき     | 自分が作業を終えて、みんなに見てほしいとき    |

#### 図解

- プル（pull）  
  GitHub → あなたのPC（最新状態が届く）

- プルリクエスト（pull request）  
  あなたの修正 → GitHub（みんな向けに「追加して良い？」と申請）
